# SOP การใช้งานระบบ KPS Transportation ERP

**สำหรับ: พนักงาน / ผู้จัดการ / ผู้ดูแลระบบ (แอดมิน)**

> เอกสารนี้เป็นคู่มือการใช้งานระบบโดยรวมตามบทบาทผู้ใช้
> สำหรับขั้นตอนละเอียดของ **งานขนส่ง (Dispatch Round)** และ **รอบน้ำมัน (Fuel Round)** ดูที่ [`SOP.md`](../SOP.md)

---

## 1. ภาพรวมระบบและบทบาทผู้ใช้

ระบบแบ่งผู้ใช้เป็น 3 บทบาท:

| บทบาท | ชื่อในระบบ | ขอบเขตการใช้งาน |
|--------|-----------|-----------------|
| **พนักงาน** (คนขับ/ธุรการ) | driver / EMPLOYEE | บันทึกงานประจำวัน: เปิด-ปิดงานขนส่ง, คีย์น้ำมัน, บันทึกค่าใช้จ่าย, ดูข้อมูลรถ — **ไม่เห็น** หน้ารายงาน/ข้อมูลการเงิน |
| **ผู้จัดการ** | manager | เห็นทุกเมนูปฏิบัติงาน + Dashboard + รายงาน + การเงิน + ข้อมูลพนักงาน — ยกเว้นเมนู "ตั้งค่า" และ "Admin Panel" |
| **ผู้ดูแลระบบ (แอดมิน)** | admin | เข้าถึงได้ทุกเมนู รวมถึงจัดการผู้ใช้/สิทธิ์, ข้อมูลบริษัท, Import ข้อมูล, Reset ข้อมูล |

### ตารางสิทธิ์เมนูตามบทบาท

| เมนู | พนักงาน | ผู้จัดการ | แอดมิน |
|------|:-------:|:--------:|:------:|
| Dashboard | ✗ | ✓ | ✓ |
| จัดการรถ (ดูรายการ/รายละเอียด) | ✓ (ขอแก้ไขได้ แก้เองไม่ได้) | ✓ (แก้ไขได้) | ✓ (แก้ไข + ลบได้) |
| ข้อมูลพนักงาน | ✗ | ✓ | ✓ |
| ระบบยาง | ✓ | ✓ | ✓ |
| ระบบน้ำมัน — ภาพรวมคลัง / คีย์ด่วน / น้ำมันลอยรอผูก | ✓ | ✓ | ✓ |
| ระบบน้ำมัน — ตั้งราคารายวัน / รายงานรายเดือน / สรุปคลังรวม | ✗ | ✓ | ✓ |
| งานขนส่ง — เปิดงาน / ปิดงาน / ประวัติการวิ่งงาน | ✓ (ประวัติซ่อนคอลัมน์เงิน) | ✓ | ✓ |
| งานขนส่ง — รายงานสรุป / สรุปรายเที่ยวต่อคัน / วางบิลรายลูกค้า / จัดการสถานที่ | ✗ | ✓ | ✓ |
| รถรับจ้างร่วม | ✓ | ✓ | ✓ |
| ค่าใช้จ่าย — บันทึก / สต๊อคคลัง / ทะเบียนร้านค้า-ช่าง | ✓ | ✓ | ✓ |
| ค่าใช้จ่าย — สถานะการเงิน / รายงานสรุป | ✗ | ✓ | ✓ |
| การเงิน (P&L รายคัน / ปิดงวดบัญชี) | ✗ | ✓ | ✓ |
| ตั้งค่า (ผู้ใช้งาน / ข้อมูลบริษัท / บัญชีธนาคาร) | ✗ | ✗ | ✓ |
| Admin Panel (ACL / Import / Reset) | ✗ | ✗ | ✓ |

> ถ้าเข้าหน้าที่ไม่มีสิทธิ์ ระบบจะแสดง "บัญชีของคุณไม่มีสิทธิ์ดูหน้านี้ — กรุณาติดต่อผู้ดูแลระบบ"

## 2. การเริ่มใช้งาน (ทุกบทบาท)

### 2.1 สมัครบัญชีใหม่
1. ที่หน้าเข้าสู่ระบบ กด **"สมัครสมาชิก"**
2. กรอก **ชื่อผู้ใช้** (ยาว ≥ 3 ตัว ใช้ได้เฉพาะ a-z, 0-9, `_` `.`), อีเมล, และ **รหัสผ่าน ≥ 6 ตัวอักษร** (ยืนยันให้ตรงกัน)
3. สมัครสำเร็จ → บัญชีจะอยู่ในสถานะ **"รออนุมัติ" (PENDING_APPROVAL)** — ยังเข้าใช้งานไม่ได้
4. **แจ้งแอดมิน** ให้อนุมัติบัญชีและกำหนดบทบาท (ดูข้อ 5.1)

### 2.2 เข้าสู่ระบบ
- กรอก **ชื่อผู้ใช้งาน + รหัสผ่าน** แล้วกด **"เข้าสู่ระบบ"**
- เข้าไม่ได้? เช็คว่า (1) บัญชีได้รับอนุมัติแล้วหรือยัง (2) สถานะบัญชีไม่ถูก "ปิดใช้งาน/ล็อค" — ติดต่อแอดมิน

### 2.3 เปลี่ยนรหัสผ่าน / ลืมรหัสผ่าน
- **เปลี่ยนรหัสผ่าน**: เมนูโปรไฟล์ (มุมขวาบน) → เปลี่ยนรหัสผ่าน → กรอกรหัสใหม่ ≥ 6 ตัว ให้ตรงกันทั้ง 2 ช่อง → บันทึก
- **ลืมรหัสผ่าน**: ใช้ "ลืมรหัสผ่าน" ที่หน้าเข้าสู่ระบบ หรือแจ้งแอดมินให้ส่ง **ลิงก์รีเซ็ตรหัสผ่านทางอีเมล** ให้ (ต้องมีอีเมลผูกกับบัญชี)

### 2.4 ออกจากระบบ
- กดปุ่ม **"ออกจากระบบ"** ด้านล่างของแถบเมนูซ้าย ทุกครั้งเมื่อใช้เครื่องร่วมกับผู้อื่น

## 3. SOP สำหรับพนักงาน

หน้าที่หลักของพนักงานคือ **บันทึกข้อมูลปฏิบัติงานให้ครบและถูกต้องภายในวันที่เกิดงาน**

### 3.1 งานขนส่ง (งานหลักประจำวัน)

ลำดับงานมาตรฐาน (รายละเอียดเต็มอยู่ใน [`SOP.md`](../SOP.md)):

1. **เปิดงาน** — เมนู `งานขนส่ง → เปิดงานขนส่ง`: เลือกรถ, คนขับ, เลขไมล์ต้นรอบ (ระบบเติมให้จากรอบก่อน — ตรวจก่อนใช้), วันเวลาออก → ระบบสร้างรอบ `DSP-YYYYMMDD-XXX` สถานะ DRAFT
2. **เพิ่มขา (Legs)** — กด "+ เพิ่มขาใหม่" ต่อเที่ยว: เลือกประเภทขา (Outbound / Backhaul / Return), ต้นทาง-ปลายทาง, ลูกค้า, สินค้า และ **รูปแบบราคา (ต่อตัน / ต่อกก. / เหมา)**
   - ⚠️ **น้ำหนักกรอกเป็น "ตัน" เสมอ** และตรวจ preview ค่าขนส่งใต้ฟอร์มก่อนบันทึกทุกครั้ง
3. **ปิดงาน** — เมนู `งานขนส่ง → ปิดงานขนส่ง`: กรอกเลขไมล์ปลาย, เวลากลับ, น้ำมันรวม (ลิตร), น้ำหนักปลายทางจริงจากใบชั่งรายขา, เบี้ยเลี้ยง, ค่าใช้จ่ายอื่น (ทางด่วน/ค่าจอด)
   - 🚨 น้ำหนักหายต้อง **≤ 100 กก./ขา** ไม่งั้นระบบไม่ให้ปิดรอบ — ตรวจใบชั่งต้น-ปลายก่อน
4. **ตรวจย้อนหลัง** — เมนู `งานขนส่ง → ประวัติการวิ่งงาน` (สำหรับพนักงาน คอลัมน์ที่เป็นเงินจะถูกซ่อน)

**กติกา:** ปิดรอบให้เสร็จภายในวันที่กลับถึงฐาน / อย่าเปิดรอบใหม่ค้างซ้อนโดยไม่ปิดรอบเก่า

### 3.2 คีย์น้ำมัน

มี 3 ช่องทางตามสถานการณ์:

| สถานการณ์ | เมนู | วิธีใช้ |
|-----------|------|---------|
| เติมน้ำมันทั่วไป (เคสส่วนใหญ่) | `ระบบน้ำมัน → คีย์ด่วนน้ำมัน` | กรอก วันที่, ทะเบียนรถ, แหล่งเติม (**ถังโรงงาน** หรือ **ปั๊มภายนอก**), จำนวนลิตร (> 0), ราคา/ลิตร, เลขไมล์ตอนเติม |
| คีย์น้ำมันไว้แต่ยังไม่รู้ว่าเป็นของรอบงานไหน | `ระบบน้ำมัน → น้ำมันลอยรอผูก` | รายการที่ยังไม่ผูกรอบจะค้างอยู่ที่นี่ — กด **"ผูกรอบ"** เพื่อเชื่อมกับใบงานเมื่อรู้รอบแล้ว **อย่าปล่อยค้างข้ามสัปดาห์** |
| เติมหลายครั้งในรอบเดียว (เต็มถัง → เติมกลางทาง → เต็มถังคืน) | รอบน้ำมันละเอียด (Fuel Round) | ดูขั้นตอนเต็มใน [`SOP.md`](../SOP.md) ข้อ 3 |

### 3.3 บันทึกค่าใช้จ่าย (ซ่อม/อะไหล่/บริการ)

เมนู `ค่าใช้จ่าย → บันทึกค่าใช้จ่าย`:
1. **เลือกรถ** และ **ช่าง/ร้านค้า** (บังคับ — ถ้าร้านไม่มีในระบบ เพิ่มก่อนที่ `ทะเบียนร้านค้า/ช่าง` หรือแจ้งผู้จัดการ)
2. เพิ่มรายการอย่างน้อย 1 รายการ เลือกหมวด: **ค่าบริการ / อะไหล่ / ยาง / น้ำมัน / อื่นๆ / คลังอะไหล่ KPS**
3. ตรวจยอดรวม → กดบันทึก
- ถ้าเบิกของจากคลังบริษัท ใช้หมวด "คลังอะไหล่ KPS" เพื่อให้ตัดสต๊อคที่ `ค่าใช้จ่าย → สต๊อคคลัง KPS` ถูกต้อง

### 3.4 ข้อมูลรถ — ดูได้ แก้เองไม่ได้

- ดูรายการรถ/รายละเอียดรถได้ที่เมนู `จัดการรถ`
- พบข้อมูลรถผิด (เช่น เลขไมล์, ทะเบียน, ประเภท): กดขอแก้ไข → ระบบให้กรอก **เหตุผลในการขอแก้ไข** → คำขอถูกส่งให้ **ผู้จัดการ/แอดมินอนุมัติ** ก่อนจึงมีผล
- **ห้าม** แก้ข้อมูลอ้อมระบบหรือจดไว้นอกระบบแทนการส่งคำขอ

### 3.5 สิ่งที่พนักงาน "ทำไม่ได้" (โดยออกแบบ)

- ไม่เห็น Dashboard, รายงานสรุป, หน้าการเงิน, ข้อมูลราคา/บิลลูกค้า, ข้อมูลพนักงานคนอื่น
- ลบข้อมูล/แก้ข้อมูลหลัก (รถ, สถานที่, ราคาน้ำมันรายวัน) ไม่ได้
- ถ้าจำเป็นต้องแก้รายการที่บันทึกผิดและระบบไม่ให้แก้ → **แจ้งผู้จัดการหรือแอดมิน** พร้อมรหัสรอบงาน (DSP-xxx / FRD-xxx) และรายละเอียดที่ถูกต้อง

## 4. SOP สำหรับผู้จัดการ

ผู้จัดการทำได้ทุกอย่างที่พนักงานทำได้ และเพิ่มหน้าที่กำกับดูแล:

### 4.1 งานประจำวัน
- **ตรวจ Dashboard** ทุกเช้า: ภาพรวมงานค้าง รอบที่ยังไม่ปิด และตัวเลขผิดปกติ
- **อนุมัติคำขอแก้ไขข้อมูลรถ** ที่พนักงานส่งเข้ามา — ตรวจเหตุผลก่อนอนุมัติทุกครั้ง
- **ตั้งราคาน้ำมันรายวัน** (`ระบบน้ำมัน → ตั้งราคารายวัน`) ให้เสร็จก่อนพนักงานเริ่มคีย์งานของวัน
- ตรวจ `ระบบน้ำมัน → น้ำมันลอยรอผูก` — ไล่ปิดรายการที่ค้างผูกนานเกิน 7 วัน

### 4.2 งานรายสัปดาห์ / รายเดือน
- **รายงานสรุปงานขนส่ง** (`งานขนส่ง → รายงานสรุป`, `สรุปรายเที่ยวรายเดือน (ต่อคัน)`) — ตรวจ km/L และกำไรต่อรอบ หาค่าผิดปกติ
- **วางบิลลูกค้า** (`งานขนส่ง → สรุป/วางบิลรายลูกค้า`) — สรุปยอดค้างรับต่อลูกค้า
- **รายงานน้ำมันรายเดือน / สรุปคลังน้ำมันรวม** — กระทบยอดน้ำมันคงคลังกับการเบิกจริง
- **ค่าใช้จ่าย → สถานะการเงิน / รายงานสรุป** — ติดตามยอดค้างจ่ายร้านค้า/ช่าง และกดบันทึกเมื่อชำระเงินแล้ว
- **การเงิน → P&L รายคัน** — ดูกำไรขาดทุนต่อคัน
- **การเงิน → ปิดงวดบัญชี** — ปิดงวดเมื่อข้อมูลของเดือนครบถ้วนแล้วเท่านั้น (รอบที่ยังค้าง DRAFT ต้องปิดหรือยกเลิกก่อน)

### 4.3 ข้อมูลหลัก (Master Data)
- ดูแล `งานขนส่ง → จัดการสถานที่`, `ข้อมูลพนักงาน`, `ระบบยาง`, และ `รถรับจ้างร่วม → คนขับรถร่วม` ให้เป็นปัจจุบัน — ข้อมูลหลักผิดจะทำให้รายงานทั้งระบบเพี้ยน

## 5. SOP สำหรับแอดมิน

แอดมินทำได้ทุกอย่างข้างต้น และรับผิดชอบส่วน **ตั้งค่า** กับ **Admin Panel**

### 5.1 อนุมัติผู้ใช้ใหม่และจัดการบัญชี

**เมนู:** `ตั้งค่า → จัดการผู้ใช้งาน` (หรือ `Admin Panel → จัดการผู้ใช้ + ACL`)

ขั้นตอนเมื่อมีคนสมัครใหม่:
1. บัญชีใหม่จะขึ้นสถานะ **"รออนุมัติ"** (มีตัวเลขแจ้งเตือนจำนวนที่ค้าง)
2. **ยืนยันตัวตนก่อนอนุมัติ** — ตรวจว่าเป็นพนักงานจริงของบริษัท
3. กด **"อนุมัติ"** → สถานะเปลี่ยนเป็น "ใช้งาน" (ACTIVE)
4. **กำหนดบทบาท** ให้เหมาะกับหน้าที่: พนักงาน / ผู้จัดการ / ผู้ดูแลระบบ — ให้สิทธิ์ต่ำสุดที่พอทำงานได้ (least privilege) และให้มีแอดมินเท่าที่จำเป็น

การจัดการบัญชีอื่นๆ ในหน้าเดียวกัน:
- **ปิดใช้งาน (INACTIVE)** — ใช้เมื่อพนักงานลาออก/พักงาน (แนะนำให้ใช้แทนการลบ เพื่อคงประวัติ)
- **เปิดใช้งานคืน** — สำหรับบัญชีที่ถูกปิด
- **ลบบัญชี** — ใช้เฉพาะบัญชีสมัครผิด/ซ้ำที่ยังไม่เคยมีข้อมูลงาน
- **ส่งลิงก์รีเซ็ตรหัสผ่าน** — ส่งอีเมลรีเซ็ตให้ผู้ใช้ที่ลืมรหัสผ่าน (บัญชีต้องมีอีเมล)
- ℹ️ ระบบไม่ให้แก้บทบาท/สถานะของ **บัญชีตัวเอง** — ต้องให้แอดมินอีกคนทำ

### 5.2 กำหนดสิทธิ์รายโมดูล (ACL Override)

**เมนู:** `Admin Panel → จัดการผู้ใช้ + ACL`

- นอกจากบทบาทหลัก สามารถกำหนด **สิทธิ์เพิ่มเติม (override) รายโมดูล** ให้ผู้ใช้รายคน: ดู (VIEW) / แก้ไข (EDIT) / ลบ-อนุมัติ (DELETE_APPROVE)
- ใช้เมื่อผู้ใช้ต้องการสิทธิ์เกินบทบาทปกติเฉพาะบางโมดูล (เช่น ธุรการที่ต้องเห็นรายงานน้ำมัน) — ไม่ต้องยกระดับเป็นผู้จัดการทั้งบัญชี
- ทุกการเปลี่ยนสิทธิ์ถูกบันทึกใน **Audit Log** — ตรวจย้อนหลังได้ในหน้าเดียวกัน

### 5.3 ข้อมูลบริษัทและบัญชีธนาคาร

**เมนู:** `ตั้งค่า → ข้อมูลบริษัท` และ `ตั้งค่า → บัญชีธนาคารบริษัท`
- อัปเดตชื่อ/ที่อยู่/เลขผู้เสียภาษีของบริษัท และบัญชีธนาคารที่ใช้ออกบิล — ข้อมูลส่วนนี้ถูกใช้ในเอกสาร/ใบวางบิล ตรวจให้ถูกต้องก่อนรอบวางบิลทุกเดือน

### 5.4 นำเข้าข้อมูล (Import File)

**เมนู:** `Admin Panel → Import File`
1. เตรียมไฟล์ Excel (`.xlsx` / `.xls` / `.csv`) ตามหัวตารางที่ระบบกำหนด (ระบบจะอ่านชีต "รถร่วม" ก่อน ถ้าไม่มีจะใช้ชีตแรก)
2. กด **"เลือกไฟล์ Excel"** → ระบบแสดงตัวอย่างจำนวนแถว/กลุ่ม และ **คำเตือน (warnings)** ของแถวที่มีปัญหา
3. **อ่าน warnings ให้หมดก่อนกดนำเข้า** — แก้ไฟล์ต้นทางแล้วอัปโหลดใหม่ถ้าจำเป็น
4. นำเข้าแล้วสุ่มตรวจข้อมูลในหน้าโมดูลจริงอีกครั้ง

### 5.5 ล้างข้อมูลระบบ (Reset Data) ⚠️ อันตรายสูงสุด

**เมนู:** `Admin Panel → Reset Data`

> **ข้อมูลที่ลบแล้วกู้คืนไม่ได้** — ใช้เฉพาะกรณีเริ่มระบบใหม่/ล้างข้อมูลทดสอบ และต้องได้รับความเห็นชอบจากเจ้าของกิจการก่อนทุกครั้ง

ขั้นตอนบังคับ:
1. **เลือกขอบเขตข้อมูลที่จะลบ**: ทั้งหมด / ข้อมูลหลัก / ค่าใช้จ่าย / ทริป / น้ำมัน / ยาง / สต็อค ("ทั้งหมด" จะเหลือเพียงบัญชีผู้ใช้ + ตั้งค่า)
2. **ตรวจสอบรายการที่จะถูกลบ** ในขั้นยืนยันของระบบ
3. **ดาวน์โหลดสำรองข้อมูล (backup) ก่อนลบทุกครั้ง — ห้ามข้าม** และเก็บไฟล์ backup ไว้นอกเครื่องที่ใช้งาน
4. พิมพ์คำว่า **`RESET`** (ตัวพิมพ์ใหญ่) เพื่อยืนยัน แล้วจึงกดลบ
5. ตรวจผลใน **ประวัติการรีเซ็ต** (`Admin Panel → Reset Data → ประวัติ`) — ระบบบันทึกผู้ทำ วันเวลา และสถานะสำเร็จ/ล้มเหลวทุกครั้ง

## 6. วินัยการใช้ระบบ (ทุกบทบาท)

1. **บันทึกภายในวันเดียวกับที่เกิดงาน** — ข้อมูลย้อนหลังคือสาเหตุหลักของตัวเลขเพี้ยน
2. **1 คน 1 บัญชี** — ห้ามใช้บัญชีร่วมกัน ห้ามบอกรหัสผ่านผู้อื่น (Audit log ผูกกับบัญชี)
3. ตรวจ **preview/ยอดรวม** ก่อนกดบันทึกทุกครั้ง — แก้ก่อนบันทึกง่ายกว่าแก้ทีหลัง
4. เจอข้อมูลผิดที่ตัวเองแก้ไม่ได้ → แจ้งผู้จัดการ/แอดมินทันที พร้อมรหัสอ้างอิง (DSP-xxx, FRD-xxx)
5. ออกจากระบบเมื่อใช้เครื่อง/มือถือร่วมกับผู้อื่น
