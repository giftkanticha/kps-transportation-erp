# คู่มือการใช้งาน — งานขนส่ง & รอบน้ำมัน (SOP)

> คู่มือฉบับนี้เจาะลึกเฉพาะ workflow งานขนส่งและรอบน้ำมัน
> สำหรับ SOP การใช้ระบบโดยรวมตามบทบาท (พนักงาน / ผู้จัดการ / แอดมิน) ดูที่ [`docs/SOP-system-usage.md`](./docs/SOP-system-usage.md)

## 1. ภาพรวม — ใช้ระบบไหนเมื่อไหร่?

ระบบมี 2 ส่วนที่เกี่ยวกับการบันทึกการวิ่งงาน — แยกหน้าที่กันชัดเจน:

### A. "งานขนส่ง" (Dispatch Round)
ใช้บันทึก **รอบขับงานของรถ** ตั้งแต่ออกจากฐาน → ส่งสินค้า (หลายขาก็ได้) → กลับฐาน
- 1 รอบ = 1 ทริปออก-กลับของรถ 1 คัน
- ภายในรอบมี **"ขา" (leg)** ได้หลายขา = หลายเที่ยว/หลายลูกค้า
- บันทึก: เลขไมล์ต้น-ปลาย, ค่าขนส่งต่อขา, น้ำมันรวม, เบี้ยเลี้ยง, ค่าใช้จ่ายอื่น
- คำนวณ: ระยะทาง, km/L, รายรับ, ต้นทุน, กำไรของรอบ

### B. "รอบน้ำมัน" (Fuel Round) — เมนู ระบบน้ำมัน
ใช้เมื่อต้อง **เติมน้ำมันหลายครั้ง** ในรอบงานเดียว (เช่น เติมเต็มถังก่อนออก + เติมปั้มนอกระหว่างทาง + เติมเต็มถังคืนตอนกลับ)
- ทำงาน 3 stage: **เปิด (start full)** → **เติมระหว่างทาง (intermediate)** → **ปิด (end full)**
- คำนวณ "น้ำมันใช้จริง" ได้แม่นยำกว่า เพราะ track ตั้งแต่ถังเต็ม → ถังเต็ม
- ผูกกับรอบงานขนส่งก็ได้ ไม่ผูกก็ได้

> **เคสปกติ (80%)**: ใช้แค่ "งานขนส่ง" → กรอกน้ำมันรวมตอนปิดงาน
>
> **เคสซับซ้อน (20%)**: เปิด "รอบน้ำมัน" คู่กัน เพื่อบันทึกการเติมแต่ละครั้ง

## 2. SOP — งานขนส่ง (Dispatch Round)

### ขั้นที่ 1: เปิดงาน
**เมนู:** `งานขนส่ง → เปิดงานขนส่ง`

กรอกข้อมูลพื้นฐานของรอบ:

| ช่อง | คำอธิบาย | ตัวอย่าง |
|------|---------|---------|
| **เลือกรถ** | ทะเบียนรถที่จะใช้ | บค-9876 |
| **คนขับ** | คนที่ขับรอบนี้ | วิชัย |
| **เลขไมล์ต้นรอบ** | ระบบเติมจากไมล์ปลายของรอบก่อนให้อัตโนมัติ (แก้ได้) | 102,500 km |
| **วันที่/เวลาออก** | เวลาออกจากฐาน | 14/04/2568 08:00 |
| **หมายเหตุ** | บันทึกเพิ่ม (ถ้ามี) | "ส่งด่วน" |

กดปุ่ม **"สร้างรอบและจัดการขา →"** → ระบบจะสร้างรอบรหัส `DSP-YYYYMMDD-XXX` สถานะ DRAFT แล้วพาไปหน้าจัดการขา

---

### ขั้นที่ 2: เพิ่มขา (Legs)
หน้าจัดการขา แสดงข้อมูลรอบและตารางขา

กดปุ่ม **"+ เพิ่มขาใหม่"** จะเปิด popup กรอกข้อมูลขา:

#### 2.1 ประเภทขา
- **Outbound** — เที่ยวขนสินค้าไป (มีรายได้)
- **Backhaul** — เที่ยวขนสินค้ากลับ (มีรายได้)
- **Return** — เที่ยวกลับเปล่า (ไม่มีรายได้, ไม่ต้องกรอกน้ำหนัก/ราคา)

#### 2.2 ข้อมูลขา
- **ต้นทาง / ปลายทาง** (ชื่อจังหวัด/สถานที่)
- **ลูกค้า** — เลือกจากรายการลูกค้า
- **ประเภทสินค้า / รายละเอียดสินค้า**

#### 2.3 รูปแบบราคา ⚠️ สำคัญมาก
เลือกให้ตรงกับการคิดเงินของลูกค้า:

| รูปแบบ | สูตรคำนวณ | ตัวอย่าง |
|--------|-----------|----------|
| **ต่อตัน** (per_ton) | น้ำหนัก (ตัน) × ราคา (฿/ตัน) | 25 ตัน × 1,800 ฿/ตัน = **45,000 ฿** |
| **ต่อกิโลกรัม** (per_kg) | น้ำหนัก (ตัน) × 1,000 × ราคา (฿/กก.) | 25 ตัน × 1,000 × 0.50 ฿/กก. = **12,500 ฿** |
| **เหมา** (lump) | ราคาคงที่ — ไม่อิงน้ำหนัก | เหมา **8,000 ฿** |

> 💡 **น้ำหนักกรอกเป็น "ตัน" เสมอ** ไม่ว่าจะเลือกรูปแบบราคาแบบไหน
>
> ใต้ฟอร์มจะมี preview แสดงค่าขนส่งที่คำนวณแบบสด — ตรวจให้แน่ใจก่อนกดบันทึก

กด **"บันทึก"** → ขาจะถูกเพิ่มในตาราง ทำซ้ำสำหรับขาที่ 2, 3, ...

---

### ขั้นที่ 3: ปิดรอบ
**เมนู:** `งานขนส่ง → ปิดงานขนส่ง`

จะเห็นรายการรอบงาน DRAFT ทั้งหมด — คลิกรอบที่ต้องการปิด เพื่อเข้าหน้าฟอร์มปิด

กรอก:
1. **เลขไมล์ปลายรอบ** (km) — ระบบจะคำนวณระยะทางอัตโนมัติ
2. **เวลากลับถึงฐาน**
3. **น้ำมันรวมในรอบ (ลิตร)** — ระบุรวมทุกการเติม (ถ้ามี Fuel Round จะ sync จากตรงนั้น)
4. **ต่อรายขา**:
   - **น้ำหนักปลายทาง (ตัน)** — น้ำหนักจริงที่ส่งถึง (จากใบชั่ง)
     - 🔄 ระบบจะ **recalc ค่าขนส่งใหม่** ตามน้ำหนักนี้ (สำหรับ per_ton / per_kg เท่านั้น; "เหมา" ไม่เปลี่ยน)
     - 🚨 **น้ำหนักหายต้อง ≤ 100 กก. ต่อขา** — ถ้าเกิน ระบบจะเตือนสีแดง **ห้ามปิดรอบ**
   - **เบี้ยเลี้ยงขานี้** — เงินที่จ่ายคนขับสำหรับขานี้
   - **หมายเหตุขา**
5. **ค่าใช้จ่ายอื่นๆ** (ทางด่วน, ค่าจอด ฯลฯ) — กด "+ เพิ่มรายการ"
6. **หมายเหตุรอบ**

กดปุ่ม **"ปิดรอบ"** → ระบบสรุปรอบและเปลี่ยนสถานะเป็น **CLOSED**

---

### ขั้นที่ 4: ดูสรุป
- **รายงานสรุป** (`งานขนส่ง → รายงานสรุป`): รวมทุกรอบช่วงเวลาที่เลือก
- **ประวัติการวิ่งงาน** (`งานขนส่ง → ประวัติการวิ่งงาน`): ดูรอบที่ปิดแล้ว, ค้นหา, kpi เช่น km/L

## 3. SOP — รอบน้ำมัน (Fuel Round) [ใช้เมื่อจำเป็น]

### เมื่อไหร่ที่ต้องเปิดรอบน้ำมัน?
ใช้เมื่อ:
- ✅ มีการเติมน้ำมันหลายครั้งในรอบเดียว (เริ่ม + กลาง + ปิด)
- ✅ ต้องการบันทึกค่าน้ำมันต่อปั้มแบบละเอียด
- ✅ ต้องการ track น้ำมันใช้จริง ตั้งแต่ "เต็มถัง" → "เต็มถังครั้งถัดไป"

ถ้าเติมแค่ครั้งเดียวหรือใช้น้ำมันจากคลังในบริษัทเท่านั้น — ไม่ต้องเปิดรอบน้ำมัน (กรอกน้ำมันรวมตอนปิดงานก็พอ)

---

### ขั้นที่ 1: เปิดรอบ
**เมนู:** `ระบบน้ำมัน → เปิดรอบน้ำมัน (ละเอียด)`

กรอก:
- **รถ, คนขับ**
- **รอบงานขนส่งที่ผูก** (เลือกจาก DRAFT — ถ้ามี) — เชื่อม fuel round กับ dispatch round
- **เลขไมล์ต้น**
- **เติมเริ่มต้น (เติมเต็มถัง)**:
  - สถานที่ (ปั้ม) เช่น "ปตท. สาขามวกเหล็ก"
  - จำนวนลิตร, ราคา/ลิตร, ราคารวม
  - เวลาเติม

ระบบสร้างรอบรหัส `FRD-YYYYMMDD-XXX` สถานะ **OPEN**

---

### ขั้นที่ 2: เติมระหว่างทาง (ถ้ามี)
**เมนู:** `ระบบน้ำมัน → เติมปั้มนอก`

ทุกครั้งที่เติมระหว่างทาง (ปั้มนอกระบบ):
- เลือกรอบน้ำมันที่ OPEN
- เลขไมล์ขณะเติม
- สถานที่, ลิตร, ราคา, เวลา
- กดบันทึก → จะเพิ่มในรายการเติมของรอบนี้

ทำซ้ำได้ทุกครั้งที่เติม

---

### ขั้นที่ 3: ปิดรอบ
**เมนู:** `ระบบน้ำมัน → ปิดรอบน้ำมัน`

- เลือกรอบน้ำมันที่ OPEN
- **เลขไมล์ปลาย**
- **เติมตอนปิด (เติมเต็มถังคืน)** ⚠️ สำคัญ! ต้องเติมจนเต็มอีกครั้ง
  - สถานที่, ลิตร, ราคา

ระบบจะคำนวณ:
- **น้ำมันใช้จริง** = ผลรวมเติม (เริ่ม + กลาง + ปิด)
- **ระยะทาง** = ไมล์ปลาย − ไมล์ต้น
- **km/L** = ระยะทาง ÷ น้ำมันใช้

> 📌 **หลักการ:** รอบน้ำมัน 1 รอบ track ตั้งแต่ "เริ่มเต็มถัง" → "เต็มถังครั้งถัดไป"
> เพราะถังที่เต็มทั้ง 2 ด้าน ทำให้ผลรวมการเติม = น้ำมันที่ใช้พอดี

---

## 4. ตัวอย่างการใช้งานจริง

### เคส 1: รอบง่ายๆ — 1 ขา, ไม่มีปั้มนอก
```
1. เปิดงาน DSP-20250414-001 (รถ, คนขับ, ไมล์ต้น 100,000)
2. เพิ่มขา 1: นครราชสีมา → ขอนแก่น
   - ข้าวสาร 25 ตัน, รูปแบบ "ต่อตัน", ราคา 1,800 ฿/ตัน → 45,000 ฿
3. ปิดงาน (ไมล์ปลาย 100,250 → 250 km / น้ำมัน 40L / น้ำหนักปลาย 24.95 ตัน)
   → ค่าขนส่ง recalc: 24.95 × 1,800 = 44,910 ฿
   → น้ำหนักหาย 50 กก. (≤ 100, ผ่าน) ✓
```

### เคส 2: รอบ multi-leg — 2 ขา ไป-กลับมีสินค้า
```
1. เปิดงาน DSP-20250414-002
2. เพิ่มขา 1 (Outbound): นครราชสีมา → ขอนแก่น
   - ข้าวสาร 25 ตัน, ต่อตัน 1,800 ฿/ตัน → 45,000 ฿
3. เพิ่มขา 2 (Backhaul): ขอนแก่น → นครราชสีมา
   - มันสำปะหลัง 20 ตัน, ต่อตัน 450 ฿/ตัน → 9,000 ฿
4. ปิดงาน
   - กรอกน้ำหนักปลายทั้ง 2 ขา (recalc รายรับใหม่)
   - น้ำมันรวมตลอดรอบ, เบี้ยเลี้ยง
```

### เคส 3: รอบที่ต้องเติมปั้มนอก
```
1. เปิดงานขนส่ง DSP-xxx
2. เปิดรอบน้ำมัน FRD-xxx (ผูกกับ DSP) + เติมเริ่ม 200L
3. เพิ่มขาในงานขนส่งให้ครบ
4. ระหว่างทาง: เติมปั้มนอก 50L (บันทึกผ่านเมนู "เติมปั้มนอก")
5. กลับฐาน: ปิดรอบน้ำมัน + เติมปิด 60L
   → น้ำมันใช้รวม = 200 + 50 + 60 = 310 L
6. ปิดงานขนส่ง — น้ำมัน sync จาก fuel round
```

## 5. ข้อควรระวัง / Pitfalls

| ปัญหา | สาเหตุ / วิธีแก้ |
|------|------------------|
| **น้ำหนักหายเกิน 100 กก.** | ปิดรอบไม่ได้ — ตรวจใบชั่งต้น-ปลาย หรือบันทึกการสูญเสีย (ระเหย/หกหล่น) |
| **รูปแบบราคาเลือกผิด** | ค่าขนส่งจะเพี้ยน — ดู preview สดใต้ฟอร์มก่อนกดบันทึก |
| **น้ำหนักหน่วยผิด** | กรอกเป็น "**ตัน**" เสมอ ไม่ว่าจะ per_ton หรือ per_kg |
| **เลขไมล์ปลาย ≤ ต้น** | ปิดรอบไม่ได้ — ตรวจเลขไมล์ |
| **ปิดรอบไม่ได้ — ไม่มีขา** | ต้องเพิ่ม Outbound/Backhaul อย่างน้อย 1 ขาก่อน |
| **น้ำหนักปลายไม่ต้องกรอก** | สำหรับ Return leg (เที่ยวเปล่า) หรือรูปแบบราคา "เหมา" |
| **รอบน้ำมันต้องเริ่ม-ปิดที่ "เต็มถัง"** | ถ้าไม่เต็มทั้ง 2 ด้าน การคำนวณน้ำมันใช้จะไม่ตรง |
